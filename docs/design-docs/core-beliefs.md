# Core Beliefs

1. The repository must explain itself.
2. A short map is more useful than a long instruction dump.
3. Context that is not encoded in the repo does not exist for agents.
4. Constraints should be enforceable, not merely suggested.
5. Plans are part of the product of engineering, not side notes.
6. Boring, inspectable systems outperform clever opaque ones in agent workflows.
7. Drift should be handled continuously with mechanical checks and cleanup.
